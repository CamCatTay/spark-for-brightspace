Spark for Brightspace

Privacy Policy

Effective Date: March 24, 2026

1. Overview

Spark for Brightspace is a free, open-source browser extension that surfaces your assignment due dates in a side panel while you browse your institution's Brightspace (D2L) platform. This policy explains exactly what data the extension accesses, what it stores locally, and what it never does.

The short version: your data never leaves your browser. We collect nothing, transmit nothing, and have no servers.

2. What the Extension Reads

When the extension is active on a Brightspace page, it calls your institution's own Brightspace REST API — the same API your browser uses when you navigate the site normally — to retrieve:

  • Your active course enrollments
  • Assignment (dropbox) names, due dates, and submission links
  • Quiz names, start dates, due dates, and end dates
  • Discussion forum and topic names and due dates
  • General content items with due dates

This data is fetched directly from your institution's Brightspace server using your existing authenticated session. The extension never sees your login credentials.

3. What Is Stored Locally

To avoid re-fetching data every time you navigate between pages, the extension caches information locally in your browser using the standard chrome.storage.local and localStorage APIs. The following is stored:

  • Course and assignment data — a local cache of the API response described above, so the panel loads instantly on the next page.
  • Panel scroll position — so the panel returns to where you left off when switching tabs.
  • Active panel tab ID — to ensure only one tab shows the panel at a time.
  • Panel width and expanded/collapsed state — so your layout preference is remembered across page loads.

All of this data is stored only on your device and is never transmitted anywhere. You can clear it at any time by clearing your browser's extension storage or uninstalling the extension.

4. What We Never Do

  • We do not collect, transmit, or share any personal or academic data.
  • We do not run any analytics, telemetry, or usage tracking.
  • We do not integrate with any third-party services.
  • We do not have a backend server or database.
  • We do not access any data outside of your institution's Brightspace domain.

5. Permissions

The extension requests the following browser permissions:

  • tabs — used to detect which tab is viewing Brightspace and to coordinate the single-panel behavior across multiple open tabs.
  • scripting — used to inject the side panel UI into Brightspace pages.
  • storage — used to cache course data and save your panel preferences locally.

The extension's content scripts only activate on pages matching https://*/d2l/* — it is completely inactive on all other websites.

6. Open Source

Spark for Brightspace is fully open source. You can inspect every line of code in the GitHub repository to verify these claims for yourself.

7. Changes to This Policy

If this policy changes, the updated version will be committed to the repository and the effective date above will be updated. Significant changes will be noted in the release notes.
